## 1. 最终结果

- 完成状态：已完成。
- 用户可见结果：
  - 属性弹窗显示后增加全窗口事件层，点击、右键和滚轮不会继续投递到底层文件管理器。
  - 属性弹窗标题区域可拖拽移动，窗口大小变化后弹窗坐标会钳制在可见区域内。
  - 属性弹窗关闭按钮改为 `icons/close.svg`，复用主窗口关闭按钮尺寸、padding 和 hover 效果。
  - 属性弹窗概要和权限页的信息改为水平左对齐、垂直居中；通用属性行使用固定标签宽度和值左对齐。
  - 权限页可修改当前文件夹 owner/group/other 访问位；点击访问行只修改待保存 mode，点击“更改”后后台保存并刷新属性。
  - 权限页移除“更改内容文件的权限”入口和内容权限页，底部改为“取消”和“更改”两个操作按钮；取消会放弃当前待保存权限改动。
- 非目标保持：
  - 不修改 owner/group。
  - 不递归修改内容权限，也不展示递归内容权限设置页。
  - 不处理 ACL、扩展属性或桌面服务集成。

## 2. 关键改动

- `filesystem-core`
  - 新增 `set_permissions(path, mode)`，校验 mode 不超过 `0o7777`，通过 Rust 标准库修改权限。
  - 测试从 17 个增加到 19 个，覆盖临时目录权限修改和非法 mode 拒绝。
- `filesystem-gui`
  - `PropertiesDialog` 新增位置、待保存权限 mode、保存中标记和错误文本。
  - GUI 新增窗口尺寸、属性弹窗指针和拖拽状态。
  - `properties_overlay` 改为按坐标布局，并用全窗口 `mouse_area` 捕获 press/release/right/scroll。
  - 权限页新增访问级别循环、取消待保存改动、后台保存和保存后重新读取属性。
  - GUI 测试从 8 个增加到 11 个，覆盖属性弹窗坐标钳制和权限位切换 helper。
- 文档：
  - 新增编号 3 research/plan/summary。
  - 更新产品概览和开发概览。

## 3. 安全门禁结果

| 项 | 结论 |
|----|------|
| 风险矩阵 | L4，涉及文件系统权限写操作和 GUI 状态流转 |
| 命令权限 | C1，只修改源码/文档并运行本地构建测试 |
| 高风险项 | 权限修改通过 core API 限制 mode，自动化测试只修改临时目录 |
| 破坏性操作 | 未对真实用户目录执行测试性 chmod/chown |
| 副作用/风险 | 运行时用户在权限页点击“更改”会修改当前文件夹本身权限 |

## 5. 审视结果

- 安全：权限修改只通过 core `set_permissions` 执行，mode 超过 `0o7777` 会拒绝；自动化测试只修改临时目录，未执行真实用户目录 chmod/chown。
- 产品：用户反馈点均有对应实现；内容权限相关入口已移除，权限页只修改当前文件夹本身权限。
- 架构：core 负责权限写操作，GUI 只维护待保存状态和发起后台任务；未引入新依赖或桌面服务。
- 工程：新增测试覆盖权限 mode 修改、非法 mode、弹窗位置钳制和权限位切换；真实图形会话视觉与交互仍需手工验证。

## 6. 残余风险

- 不提供内容权限页，也不递归修改文件/文件夹权限。
- 权限修改失败时只在属性弹窗中显示错误并保留待保存状态；没有额外恢复策略。
- 拖拽区域限定在标题文字区域，关闭和返回按钮区域不会触发拖拽。
