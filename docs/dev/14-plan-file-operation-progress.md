## 1. 目标与成功标准

- 任务目标：
  - 修正跨文件系统剪切语义：同文件系统剪切使用 `rename`，跨文件系统剪切复制成功后删除源路径。
  - 为复制/剪切粘贴提供整体圆形进度入口和每次粘贴详情进度。
- 成功标准：
  - 用户触发一次或多次粘贴时，侧栏标题“文件”居中显示，标题右侧显示圆形总进度；完成后中心显示对钩并延时 3 秒消失。
  - 点击圆形区域可展开/隐藏详情；详情作为窗口上层 popup 显示，不改变侧栏或主区域布局；点击 popup 外部区域会自动隐藏详情；详情不超过 5 条时不显示滚动条。
  - 取消某次操作会停止该操作，立即移除对应详情条，并清理该操作已创建但未完成的目标根路径；其它操作继续。
  - 同时最多启动 3 个文件操作复制线程，多余操作显示为排队状态，前序操作结束后再启动。
  - core 测试覆盖进度、取消和跨设备 fallback 判定；GUI 构建和相关测试通过。
- 前置条件：
  - 继续使用现有内部剪贴板和系统文本剪贴板解析。
  - 复制/移动仍不覆盖已有目标。
- 非目标：
  - 不实现暂停/恢复、重试、系统通知、专用桌面剪贴板格式或回收站。

## 4. Bug 修复计划（按需）

- 复现方式：
  - 代码阅读确认 `PasteAction::Cut` 只调用 `fs::rename`；跨文件系统会返回 `EXDEV` 并失败。
- 证据等级：E2。
- 根因假设：
  - `paste_paths` 没有识别 `EXDEV` 并进入复制后删除流程。
- 最小修复点：
  - 在带进度粘贴 API 的剪切分支中保留 `rename` 快路径，并对 `EXDEV` 调用复制后删除。
  - 原 `paste_paths` 复用新 API 的无进度路径，避免语义分叉。
- 回归/相关验证：
  - 同文件系统移动测试保持通过。
  - 增加 `EXDEV` 判定纯函数/分支测试或可注入测试。

## 6. 验证计划

- 基础验证：
  - `cargo fmt --check`
  - `cargo test -p filesystem-core`
  - `cargo check -p filesystem-gui`
  - `cargo test -p filesystem-gui`
  - `cargo test`
  - `make test`
  - `git diff --check`
- 高风险验证（按需）：
  - 错误路径：目标存在唯一命名、复制失败清理、取消清理、同文件系统移动保留。
  - 并发/异步：多个粘贴任务状态独立；同时最多 3 个 operation 启动复制线程；取消只影响对应 operation id。
  - API：旧 `paste_paths` 行为兼容，新增进度 API 不改变 core 对 GUI 的依赖方向。
- 验证环境：
  - 本地 Linux workspace；自动化文件操作只使用测试创建的临时目录。
- 不可执行验证项：
  - 未必能在当前环境自动构造真实两个不同文件系统的 GUI 跨设备剪切；用 `EXDEV` 判定逻辑和人工验证建议补充。
  - 真实 X11/Wayland 视觉效果需人工启动 GUI 验证。
- 残余风险：
  - 大量小文件预扫描会先显示准备状态，百分比在统计完成后才准确。
  - 取消和失败清理只针对本次创建的目标根路径，不尝试恢复已删除源路径。
