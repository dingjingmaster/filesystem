> - 关联需求：文件管理器所有时间格式要按系统时区显示，修复列表视图修改时间偏差。

## 1. 目标

- 要解决的问题：当前 `format_modified()` 直接把 `SystemTime` 的 Unix 秒按 UTC 手算年月日时分，导致列表视图“修改时间”和属性弹窗访问/修改/创建时间没有按本机时区显示。
- 成功标准：文件管理器所有使用 `format_modified()` 的时间显示都按系统本地时区格式化为 `YYYY-MM-DD HH:MM`；无法转换时显示 `-`。

## 2. 背景与边界

- 包含：GUI 层时间格式化函数、直接 `libc` 依赖、相关单元测试、产品/开发总览更新。
- 不包含：修改 core `SystemTime` 数据结构；增加用户可配置时区；改变显示格式模板；改文件排序或文件系统 metadata 读取。
- 关键假设：当前项目只支持 Linux 本地文件系统，系统本地时区由 C 运行时 `localtime_r`/`TZ`/系统时区数据决定。

## 6. 总结

- `format_modified()` 不再按 UTC 手算年月日，改为先把 `SystemTime` 转为 Unix 秒，再通过 `libc::localtime_r` 按系统本地时区格式化为 `YYYY-MM-DD HH:MM`。
- 列表视图“修改时间”、文件夹属性“修改时间/创建时间”和普通文件属性“访问时间/修改时间/创建时间”共用该格式化路径。
- 增加单元测试覆盖纪元前时间秒数取整、本地时间格式化成功和转换失败兜底。
