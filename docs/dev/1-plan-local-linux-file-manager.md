## 1. 目标与成功标准

- 任务目标：规划一个可独立运行的 Linux 本地图形文件管理器，运行时不依赖 DBus/GVFS/portal/XDG MIME/通知/桌面配置等系统桌面服务；允许通过配置直接调用外部二进制补充打开文件、压缩、预览等能力。
- 成功标准：
  - 核心文件系统能力只依赖 Linux 本地文件系统和进程执行能力。
  - 同一个二进制可在 X11 和 Wayland 会话下运行，不需要分别编译。
  - GUI 使用 `wgpu` 渲染，不再保留 `tiny-skia` 构建路径。
  - 外部二进制不存在时功能可降级，不影响核心浏览和文件操作。
  - 不引入 `libcosmic`、`gio`、`glib`、`gvfs`、`zbus`、`ashpd`、`notify-rust`、`xdg-mime`、`cosmic-config` 等桌面服务或桌面集成依赖。
  - 高风险文件操作有自动化测试覆盖：复制、移动、重命名、覆盖冲突、跨设备退化路径、符号链接、权限/时间戳、删除/回收站、失败清理。
- 前置条件：
  - 接受首版限定 Linux 本地文件系统。
  - 接受外部二进制由配置或探测启用，不把系统桌面服务作为能力来源。
  - 前端形态固定为 GUI，不做命令行/TUI。
- 非目标：
  - 不支持 GVFS、SFTP、SMB、MTP、网络邻居、云盘、桌面图标。
  - 不注册 `org.freedesktop.FileManager1`。
  - 不读取或修改系统 MIME 默认应用数据库。
  - 不依赖桌面通知、portal、会话服务或 COSMIC 配置。

## 2. 修改边界

- 最大修改范围：
  - 后续可在当前仓库创建新的 Rust workspace/project。
  - 模块建议：`core`、`ops`、`trash`、`config`、`external_cmd`、`ui_gui`、`tests`。
- 禁止触碰范围：
  - 不修改 `/data/source/cosmic-files` 源仓库。
  - 不执行删除、覆盖、安装依赖、联网下载、系统服务操作或真实用户目录破坏性测试。
- 影响模块/文件：
  - 当前阶段只新增本地计划文档。
  - 后续实现时新增项目源码和测试夹具。
- 依赖关系：
  - 核心层优先 Rust `std`；必要时只引入很小的 Linux/Unix 辅助依赖。
  - 前端层与核心层隔离，避免 iced 和渲染后端选择污染文件操作核心。
  - GUI 层依赖 `iced` 的 `wgpu`、`x11`、`wayland` features；不保留 `tiny-skia` 构建路径。
  - 外部二进制通过配置和 `std::process::Command` 调用，不通过桌面服务间接调用。

## 5. 推荐架构

| 层 | 职责 | 依赖策略 |
|----|------|----------|
| `core` | 路径模型、目录读取、排序、过滤、文件元数据、选择状态 | 优先 `std`；使用 `symlink_metadata`，避免跟随符号链接造成意外 |
| `ops` | 复制、移动、重命名、新建、删除、冲突处理、进度、取消 | 必须通过后台任务执行并做小步可取消检查；跨设备移动走复制后清理；真实写操作可测试 |
| `trash` | 无桌面服务的本地回收站 | 自实现轻量 trash 后端，记录原路径和删除时间；无法安全回收时要求明确确认永久删除 |
| `external_cmd` | 外部命令探测与调用 | 只执行配置白名单命令；不用 `xdg-open`、portal 或 MIME 服务 |
| `config` | 用户配置、命令映射、快捷键、默认行为 | 简单文本格式优先；避免桌面配置服务 |
| `ui_gui` | 首版图形前端 | 建议直接使用 `iced`；固定使用 `wgpu + x11 + wayland` |

GUI Cargo 依赖：

```toml
[dependencies]
iced = { version = "0.14", default-features = false, features = [
    "thread-pool",
    "svg",
    "wgpu",
    "x11",
    "wayland",
] }
```

外部二进制建议按能力分组启用：

| 能力 | 可选命令 | 降级策略 |
|------|----------|----------|
| 文件类型识别 | `file` | 不存在时按扩展名和 metadata 判断 |
| 文本预览 | `less`、`bat`、`sed` | 内置读取前 N KiB 文本 |
| 编辑 | `vi`、`nano`、用户配置编辑器 | 未配置则禁用编辑动作 |
| 压缩/解压 | `tar`、`bsdtar`、`gzip`、`xz`、`7z` | 缺命令则隐藏对应动作 |
| 打开文件 | 用户配置命令表 | 不使用 `xdg-open`，未配置则提示无处理器 |

## 7. 验证计划

- 基础验证：
  - `cargo check`
  - `cargo test`
  - 默认 wgpu 渲染构建：`cargo check`
  - 针对临时目录的集成测试，所有破坏性写操作只作用于测试目录。
- 高风险验证：
  - 删除：回收站路径、永久删除确认、删除失败后的状态一致性。
  - 移动：同设备 rename、跨设备退化、目标冲突、移动目录时的部分失败。
  - 复制：大文件分块、权限/mtime、符号链接、特殊文件拒绝策略。
  - 并发/取消：长操作取消后是否留下半成品，进度状态是否一致。
  - 错误路径：权限不足、路径不存在、目标已存在、磁盘空间不足的可恢复行为。
- 验证环境：
  - Linux 本地文件系统。
  - X11 和 Wayland 图形会话。
  - 可选外部命令存在/不存在两组环境。
- 不可执行验证项：
  - 真实跨设备移动可能需要专门挂载环境；没有环境时先用抽象/fixture 和手工验证记录。
  - 磁盘空间不足难以稳定自动化，可后置专项测试。
- 残余风险：
  - 自实现 trash 需要严格遵守路径边界，避免恢复覆盖。
  - 允许外部命令意味着配置必须防止隐式 shell 执行；应直接传 argv，不通过 shell。
  - `wgpu` 构建可能受 GPU 驱动/Mesa/Vulkan/OpenGL 栈影响；去掉 `tiny-skia` 后需要在目标环境做真实开窗验证。

## 10. 本次实现与验证记录

- 实现范围：
  - 新增 workspace：`filesystem-core` 与 `filesystem-gui`。
  - `filesystem-core` 使用 Rust `std` 和 `regex`，提供 `scan_dir`、`search_file_names`、`FileEntry`、`EntryKind`、`DirectoryListing`、`SearchResults`、`ScanOptions` 等只读目录扫描/文件名正则搜索 API。
  - `filesystem-gui` 使用 `iced`，固定启用 `wgpu`、`x11`、`wayland`、`thread-pool`、`svg`。
  - 目录扫描和文件名正则搜索通过 `Task::perform` 后台执行；GUI 使用自增请求 ID 忽略过期结果，避免连续导航/搜索时旧结果覆盖新状态。
  - 同一 GUI 构建路径同时启用 `iced/x11` 和 `iced/wayland`，由 winit 在运行时选择会话后端。
  - GUI 当前为无系统边框暗色文件管理器 shell：左侧本地导航、顶部后退/前进按钮、可编辑地址栏、视图菜单、隐藏文件开关、自绘窗口控制按钮、主区文件视图和底部状态栏。
  - 主区域支持图标视图和列表视图；地址栏右侧菜单按钮打开菜单后，可在“视图”子菜单中选择“图标视图”或“列表视图”。
  - 列表视图显示名称、大小、所有者、修改时间；所有者来自 `FileEntry.owner` 的 Linux UID，避免依赖 NSS/桌面服务。
  - 菜单按钮使用 `icons/menu.svg`，位于地址栏右侧拖拽空白区和窗口最小化按钮之间，靠近窗口控制按钮但保留间隔。
  - 隐藏文件开关已移入地址栏右侧菜单；工具栏不再单独显示隐藏文件复选框。
  - 菜单和视图子菜单通过主内容 `stack` 覆盖在文件视图上方，不改变图标/列表视图文件位置和布局。
  - 主区域文件和文件夹支持单击选中；空白区域拖拽框选可一次选中多个条目；只有双击才进入目录或触发普通文件打开提示。
  - 文件夹条目使用 `icons/folder.svg`；文件条目根据扩展名在本地图标主题中查找 SVG MIME 图标，主题没有对应 SVG 时回退到 `icons/file.svg`；图标解析随目录扫描/搜索在后台任务完成。
  - 侧边栏固定显示“主文件夹”和“根目录”；分割线下方只显示用户家目录中实际存在的常见目录：下载/图片/桌面/文档/音乐/视频，兼容中文和英文目录名。
  - 侧边栏导航项使用本地 SVG 图标：主文件夹 `icons/home.svg`、根目录 `icons/root.svg`、下载 `icons/download.svg`、图片 `icons/picture.svg`、桌面 `icons/desktop.svg`、文档 `icons/document.svg`、音乐 `icons/music.svg`、视频 `icons/videos.svg`；按钮内容左对齐、垂直居中。
  - 侧边栏顶部为空白窗口拖拽区；地址栏右侧保留空白拖拽区，双击切换最大化；右侧按钮使用本地 SVG 图标执行最小化、最大化/还原和关闭。

- 计划偏差：
  - `iced` 0.14 在关闭 default features 后需要显式启用 executor，因此当前固定启用 `iced/thread-pool`。
