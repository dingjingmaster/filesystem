> - 关联调研：docs/dev/2-research-context-menu-file-ops-properties.md

## 1. 目标与成功标准

- 任务目标：在现有 GUI 文件管理器中实现空白区右键菜单，并完成首批可后台执行的本地文件写操作和当前文件夹属性弹窗。
- 成功标准：
  - 图标视图和列表视图空白处右键弹出菜单，菜单包含指定条目和分割线，且不改变文件视图布局。
  - 新建文件/新建文件夹在当前目录创建唯一名称条目，并进入内联重命名状态；默认名称全选，编辑框至少使用 1.5 倍行高，长名称最多扩展到基础宽度 3 倍，继续输入会换行并增高；回车或点击文件视图/工具栏/侧边栏等外部区域提交，重命名不覆盖已有路径。
  - 粘贴从标准剪贴板文本解析本地绝对路径或 `file://` URI；复制递归复制文件/目录；剪切在同文件系统内用 `rename` 移动。
  - 全选会选中当前视图列出的所有条目。
  - 在终端打开按 `terminator`、`mate-terminal`、`gnome-terminal` 顺序查找 PATH 并在当前目录启动，不经 shell。
  - 属性弹窗展示当前文件夹名称、条目统计、总大小、剩余空间、父目录、修改/创建时间和权限概览；权限页为查看模式。
  - 所有可能阻塞 UI 的操作都走 iced 后台任务。
- 前置条件：
  - 接受首版粘贴不读取专用剪贴板 MIME target，只解析 iced 能读取的文本内容。
  - 接受权限弹窗首版只查看，不修改权限。
- 非目标：
  - 选中文件/文件夹右键菜单。
  - 删除/回收站/永久删除。
  - 跨设备剪切时复制后删除退化。
  - 权限修改和递归 chmod/chown。

## 2. 修改边界

- 最大修改范围：
  - `crates/filesystem-core/src/lib.rs`：新增创建、重命名、复制/移动粘贴、属性统计、剪贴板文本解析相关纯逻辑和测试。
  - `crates/filesystem-core/Cargo.toml`：必要时增加 `libc` 直接依赖，用于 Linux `statvfs` 获取磁盘剩余空间。
  - `crates/filesystem-gui/src/main.rs`：新增右键菜单状态、内联重命名状态、属性弹窗状态、后台任务消息和 UI。
  - `docs/overview-product.md`、`docs/overview-product-dev.md`、`docs/dev/README.md`、当前 `docs/dev/2-*`：同步长期行为和任务状态。
- 禁止触碰范围：
  - 不修改 `/data/source/cosmic-files`。
  - 不执行真实用户目录写入测试。
  - 不新增 DBus/GVFS/portal/XDG MIME/通知/桌面配置服务依赖。
- 影响模块/文件：
  - core 公共 API 会新增写操作与属性查询函数。
  - GUI 会新增用户可见右键菜单、属性弹窗和重命名交互。
- 依赖关系：
  - 保持 iced/wgpu/x11/wayland 现有 GUI 依赖。
  - `libc` 仅用于 Linux 本地文件系统属性统计，不引入桌面服务。

## 3. 安全门禁摘要

| 项 | 结论 |
|----|------|
| 风险矩阵结论 | L4，新系统中引入文件系统写操作和外部进程 |
| 命令权限 | C1，源码/文档修改和临时目录测试；提交、推送、真实用户目录写入测试均不执行 |
| 高风险开发门禁 | 是，文件系统写操作、递归复制、移动、错误恢复、外部进程 |
| 破坏性操作 | 产品功能含移动；自动化测试只在 `TempDir` 下进行 |
| 用户确认事项 | 无阻塞；首版粘贴和权限修改边界需交付说明 |
| 止损/回滚方案 | 回退本轮 core API、GUI 状态/UI 和文档即可；未修改持久配置或真实用户数据 |

## 6. 验证计划

- 基础验证：
  - `cargo fmt --check`
  - `cargo test -p filesystem-core`
  - `cargo check -p filesystem-gui`
  - `cargo test`
  - `make test`
  - `git diff --check`
- 高风险验证：
  - core 测试只在临时目录创建/复制/移动/重命名。
  - 覆盖目标存在时自动唯一命名，不覆盖已有文件。
  - 移动测试只验证同目录/同文件系统 `rename` 路径。
  - 目录递归复制不跟随符号链接目录；符号链接按链接复制。
  - 剪贴板解析覆盖 raw path、`file://`、`copy` 和 `cut` 标记。
- 验证环境：
  - 本地 Linux 工作区。
  - Rust/Cargo 当前环境。
- 不可执行验证项：
  - 不在真实 X11/Wayland 会话中手工验证右键菜单定位、剪贴板读取或终端开窗。
  - 不验证跨设备移动。
  - 不验证真实桌面文件管理器专用剪贴板 MIME target。
- 残余风险：
  - 外部文件管理器复制/剪切后，如果 iced 不能读到文本 fallback，首版粘贴菜单会提示剪贴板无本地路径。
  - 属性递归统计在超大目录上可能耗时较长；本轮后台执行但不实现取消/进度。
