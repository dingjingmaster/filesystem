## 1. 目标

- 要解决的问题：当用户家目录的 XDG 模板目录存在模板文件时，文件管理器空白处右键菜单应能按模板创建新文件。
- 成功标准：
  - 启动和打开空白菜单时读取 `XDG_TEMPLATES_DIR` 对应目录，并兼容 `Templates`、`Template`、`模板` fallback。
  - 空白菜单在“新建文件”“新建文件夹”后显示“新建模板文件”子菜单，子菜单列出模板文件名；后续修复中调整为菜单显示名不含最后一个扩展名。
  - 选择模板后把模板文件复制到当前目录，自动选择不冲突的文件名，并进入现有内联重命名流程。

## 2. 背景与边界

- 背景：xdg-user-dirs 使用 `$(XDG_CONFIG_HOME)/user-dirs.dirs` 记录本地化用户目录，模板目录可能是英文 `Templates`，也可能是中文 `模板`。
- 包含：解析用户级 `user-dirs.dirs` 中的 `XDG_TEMPLATES_DIR`；扫描模板目录直接子项中的普通文件；空白菜单二级菜单；模板复制；新文件重命名。
- 不包含：递归扫描模板目录；显示模板目录中的子目录；监听模板目录实时变化；修改系统级 XDG 配置。
- 关键假设：模板目录内的直接普通文件就是可用于创建的模板文件。
- 非目标：不改变已有“新建文件”“新建文件夹”“粘贴”“自定义命令”和属性菜单语义。
- 最大修改范围：`filesystem-core` 文件创建能力、`filesystem-gui` 菜单状态/任务/渲染和相关文档。
- 禁止触碰范围：不修改用户真实模板目录，不执行系统级配置写入。

## 4. 方案

- 推荐方案：
  - core 新增 `create_file_from_template(parent, template)`，校验模板是文件，按模板文件名生成唯一目标路径，复制内容和权限。
  - GUI 启动时和空白菜单打开时异步加载模板文件列表。
  - 空白菜单新增“新建模板文件”项，点击后在右侧展开模板子菜单。
  - 模板创建成功后复用 `RenameState` 和现有重命名覆盖层。
- 取舍理由：不依赖外部 `xdg-user-dir` 命令，不阻塞 UI 线程；只扫描直接文件，避免模板目录过大或子目录语义不清。
- 风险与应对：模板目录运行时变化可能在下一次右键打开或启动后才刷新；右键空白菜单打开时会主动刷新一次模板列表。

## 8. 总结

- 最终结果：空白处右键菜单在存在 XDG 模板文件时提供“新建模板文件”子菜单，选择模板后复制到当前目录并进入内联重命名；相关自动化验证已通过。
- 遗留风险：真实图形会话仍需人工确认菜单位置、点击展开和重命名焦点。
- 后续建议：如模板数量很多，再考虑模板子菜单滚动高度限制和键盘导航。
