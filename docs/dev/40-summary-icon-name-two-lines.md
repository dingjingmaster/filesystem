# 文件名行数限制轻量总结

## 1. 修改目标

- 原始需求：列表视图文件名只显示 1 行，大小/所有者/修改时间列保持固定够用宽度；图标视图中文件名最多占 2 行，名字太长时省略中间字符并保留收尾字符。
- 成功标准：列表视图根据当前名称列宽实时中间省略且不换行，侧列不随窗口宽度过度扩张；图标视图中文和混合扩展名长文件名不会溢出到第 3 行；缩略结果保留开头、结尾和扩展名。

## 2. 修改结果

- 修改文件：`crates/filesystem-gui/src/components.rs`、`crates/filesystem-gui/src/app.rs`、`crates/filesystem-gui/src/config.rs`、`crates/filesystem-gui/src/utils.rs`
- 行为变化：列表视图大小/所有者/修改时间列改为固定宽度，名称列填充剩余空间。
- 行为变化：列表视图文件名改为单行自定义 widget，在 layout 阶段使用 Iced paragraph 按名称列宽度实测候选文本宽度，超过一行宽度时使用中间省略号保留首尾字符；resize 时按 8px 宽度桶复用测量结果，降低拖动窗口时的重复文本 layout 成本。
- 行为变化：图标视图文件名标题改为两行自定义 widget，在 layout 阶段使用 Iced paragraph 按当前 renderer/font 实测候选文本排版高度，测量时使用不受显示高度裁剪的高度上限，超过两行区域时使用中间省略号保留首尾字符。
- 行为变化：图标视图标题宽度改为使用 tile padding 后的实际内层宽度测量和绘制，避免按外层 tile 宽度误判后导致第二行末尾被裁掉。
- 行为变化：中间省略分配从简单前后均分调整为偏向保留末尾扩展名，避免图标第二行看不到文件末尾字符。
- 回退项：移除上一版按“非 ASCII 字符 2 个宽度单位”估算的 `short_name()` 逻辑，普通短名工具恢复字符数缩略。
- 不适用项：非文件名 UI 文本的 `short_list_text()` 保持原有字符数截断逻辑。
- 安全门禁执行结果：未触碰用户数据、系统配置或远端环境。
- 风险矩阵结论：L1，局部 UI 显示修正。
- 涉及命令权限级别：C1，工作区代码和本地测试。
- 是否覆盖用户已有修改：否。
