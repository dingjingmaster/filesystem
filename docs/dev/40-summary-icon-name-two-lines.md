# 图标文件名最多两行轻量总结

## 1. 修改目标

- 原始需求：图标视图中文件名最多占 2 行，名字太长时省略中间字符并保留收尾字符。
- 成功标准：中文和混合扩展名的长文件名不会溢出到第 3 行；缩略结果保留开头、结尾和扩展名。

## 2. 修改结果

- 修改文件：`crates/filesystem-gui/src/utils.rs`
- 行为变化：`short_name()` 从简单按字符数截断改为按显示宽度预算截断，非 ASCII 字符按 2 个宽度单位计算，图标名预算控制在两行范围内，并继续使用中间省略号。
- 不适用项：列表视图的 `short_list_text()` 保持原有字符数截断逻辑。
- 安全门禁执行结果：未触碰用户数据、系统配置或远端环境。
- 风险矩阵结论：L1，局部 UI 显示修正。
- 涉及命令权限级别：C1，工作区代码和本地测试。
- 是否覆盖用户已有修改：否。

## 3. 验证

- 验证环境：本地开发环境
- 命令/步骤：`cargo test -p filesystem-gui short_name`；`cargo fmt --check`；`git diff --check`；`cargo test -p filesystem-gui`
- 结果：通过，覆盖中文长文件名缩略和短文件名保持原样；完整 GUI crate 145 个测试通过；格式和空白检查通过。
- 未执行验证项：未启动 GUI 做截图复核。
- 未覆盖风险：实际字体渲染宽度仍可能随系统字体略有差异，但当前预算按截图中的中文换行能力保守收敛。
- Bug 证据等级：E2，用户提供截图可复现三行溢出现象。
- Bug 修复验证：新增单元测试验证 `安得电子文档安全专项测试报告.docx` 缩略为 `安得电子文档…测试报告.docx`，显示宽度不超过两行预算。
