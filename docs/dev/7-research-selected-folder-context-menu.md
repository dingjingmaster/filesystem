# 选中文件夹右键菜单调研报告

> 文档元数据
> - 文件编号：7
> - 文档类型：research
> - 文件路径：docs/dev/7-research-selected-folder-context-menu.md
> - 文档版本：v1.0.0
> - 最后更新：2026-06-25
> - 关联需求：左键选中文件夹后，右键显示文件夹菜单，支持打开、复制、剪切、重命名、删除、终端打开和属性

## 1. 问题与边界

- 问题描述：当前 GUI 只有文件视图空白区域右键菜单；当存在选中项且指针命中条目时，右键不会弹出针对条目的菜单。
- 调研目的：补齐单个选中文件夹的上下文菜单，同时保持阻塞型文件操作在后台执行，并避免误删。
- 包含：
  - 单个已选中文件夹右键菜单。
  - 内部复制/剪切状态和粘贴优先级。
  - 剪切文件夹的视觉变淡。
  - 文件夹重命名入口复用现有重命名覆盖层。
  - 删除确认弹窗和后台递归删除。
  - 文件夹终端打开和属性查看。
- 不包含：
  - 文件右键菜单。
  - 多选文件夹批量操作。
  - 系统剪贴板 MIME target 写入。
  - 回收站语义。
- 非目标：不新增桌面服务依赖，不改变现有空白处菜单语义。
- 禁止触碰范围：不回退既有模块拆分改动；不在自动化验证中删除用户目录；不执行 `rm` 或系统级破坏性命令。

## 2. 当前证据

- 现有实现/现状：
  - `FileManager` 已有 `selected_paths`，单击条目会选中，双击目录会进入目录。
  - 右键逻辑目前用 `should_show_blank_context_menu` 判断是否显示空白菜单；选中项上右键不会显示菜单。
  - `paste_paths` 已支持目录递归复制和同文件系统移动，但 GUI 只有从系统剪贴板文本读取路径的粘贴入口。
  - core 没有删除 API。
  - 属性弹窗当前可显示指定目录的 `folder_properties`，但 GUI 入口只传当前目录。
  - 终端打开函数可以对任意 cwd 启动终端。
- 已知约束：
  - 所有可能阻塞 UI 的复制、移动、删除、属性统计、终端探测都必须走后台任务。
  - 删除前必须弹窗确认。
  - 删除是递归删除，不实现回收站。
- 关键日志/数据/用户反馈：用户明确要求菜单项和各项行为。
- Bug 证据等级：不适用，本次为功能实现。
- 证据不足项：未做真实 GUI 视觉/交互 smoke test。

## 3. 安全门禁摘要

| 项 | 结论 |
|----|------|
| 风险矩阵初判 | L4，涉及文件系统递归删除、后台任务和 GUI 交互扩展 |
| 命令权限 | C0/C1；不需要 C2/C3 |
| 高风险开发门禁 | 是，文件系统写操作和删除逻辑 |
| 破坏性操作 | 代码新增删除能力；自动化验证只在临时目录内执行 |
| 用户已有修改 | 当前工作区已有未提交模块拆分和属性页改动，本次基于现状追加，不回退 |
| 用户确认事项 | 用户已明确要求删除功能；运行时仍必须由确认弹窗二次确认 |

## 4. 候选方案

| 方案 | 核心思路 | 优点 | 风险/代价 | 适用条件 |
|------|----------|------|-----------|----------|
| A | 复用空白菜单并在选中时追加文件夹菜单项 | 改动少 | 菜单语义混杂，容易误触空白菜单操作 | 功能很少时 |
| B | 新增 `ContextMenuState` 区分空白菜单和文件夹菜单 | 菜单语义清晰，便于后续文件/多选菜单扩展 | 需要调整右键判断和 overlay | 当前需求 |
| C | 建立完整 command/action 框架 | 扩展性强 | 对当前规模过度设计 | 后续多类型、多选、多外部命令复杂化时 |

## 5. 推荐结论

- 推荐方案：采用方案 B。
- 取舍理由：
  - 明确区分空白菜单和单个文件夹菜单，避免修改空白菜单既有行为。
  - 内部剪贴板状态只服务本进程文件夹复制/剪切，保留系统剪贴板文本路径作为 fallback。
  - 删除 API 放在 core，GUI 只负责确认和后台调用。
- 需要进入 Plan 的关键约束：
  - 文件夹菜单只在右键命中“当前单个选中的目录”时显示。
  - 粘贴优先使用内部剪贴板；没有内部剪贴板时维持原系统剪贴板文本路径逻辑。
  - 剪切状态在粘贴成功或被其它复制/剪切覆盖后恢复视觉。
  - 删除确认弹窗必须阻止底层文件视图事件。
- 需要用户确认的问题：无。
- 后续验证方向：core 临时目录删除测试、GUI 编译和现有 GUI 单元测试、diff 检查；真实 GUI 人工验证另行执行。
