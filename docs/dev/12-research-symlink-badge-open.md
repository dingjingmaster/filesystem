# 软链接角标与打开行为调研报告

> 文档元数据
> - 文件编号：12
> - 文档类型：research
> - 文件路径：docs/dev/12-research-symlink-badge-open.md
> - 文档版本：v1.0.0
> - 最后更新：2026-06-25
> - 关联需求：文件/文件夹软链接显示角标，断链显示断开角标，双击有效软链接打开目标，双击断链弹窗提示。

## 1. 问题与边界

- 问题描述：现有扫描只把符号链接标为 `EntryKind::Symlink`，GUI 无法知道目标是文件还是目录，也无法区分断链，因此图标、菜单分类和双击打开行为都不能满足软链接语义。
- 调研目的：在不依赖桌面服务、不引入新依赖的前提下，给 core 和 GUI 增加可维护的软链接目标信息。
- 包含：目标类型、断链状态、目标真实路径、图标角标、有效类型菜单分类、双击打开和断链提示。
- 不包含：递归跟随目录软链接搜索、修改软链接目标、专门的软链接属性页。
- 非目标：不调用 `file`、`xdg-open`、DBus/GVFS/portal。
- 禁止触碰范围：不改删除/复制/剪切语义，文件/符号链接删除仍只删除路径本身。

## 2. 当前证据

- 现有实现/现状：`scan_dir` 使用 `symlink_metadata` 不跟随链接，`DisplayEntry` 只有 MIME 和基础图标；菜单按 `EntryKind::Directory` 区分文件夹和文件。
- 已知约束：扫描和装饰在后台任务执行；GUI 线程不能做耗时文件系统操作；有效目录软链接应像文件夹一样打开，断链应显示弹窗。
- 关键用户反馈：要求文件夹软链接和文件软链接在图标右上角显示 `icons/symbol.svg`，断链显示 `icons/symbol-disconnect.svg`；双击断链提示“软连接 xxx 损坏”。
- Bug 证据等级：不适用，属新功能。
- 证据不足项：未在真实 X11/Wayland 会话人工验证角标视觉位置。

## 4. 候选方案

| 方案 | 核心思路 | 优点 | 风险/代价 | 适用条件 |
|------|----------|------|-----------|----------|
| A | core 扫描时跟随链接读取目标类型、断链状态和 canonical path | GUI 无需在 UI 线程补 I/O；菜单/打开/图标语义一致 | `FileEntry` 公共结构增加字段 | 适合当前后台扫描模型 |
| B | GUI 双击或绘制时临时 `metadata/canonicalize` | 改 core 少 | UI 线程可能阻塞；列表、菜单和图标容易不一致 | 不适合当前约束 |

## 5. 推荐结论

- 推荐方案：采用方案 A，在 `FileEntry` 增加 `SymlinkTarget`，GUI 使用有效类型判断菜单和打开行为。
- 取舍理由：软链接目标信息属于目录条目元数据，放在 core 可复用，也符合后台任务边界。
- 需要进入 Plan 的关键约束：断链只提示不打开；复制/剪切/重命名/删除仍作用于软链接路径本身；目录软链接双击进入目标路径。
- 需要用户确认的问题：无。
- 后续验证方向：core 临时目录符号链接测试、GUI 编译与角标单测、整体 `make test`。
