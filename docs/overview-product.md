## 1. 产品定位

- 产品名称：英文 `File`，中文 `文件`。
- 目标用户：需要轻量、独立、图形化 Linux 本地文件管理器的用户。
- 核心问题：常见桌面文件管理器依赖 DBus/GVFS/portal/MIME/通知/桌面配置等服务，不适合独立环境运行。
- 核心价值：提供只依赖 Linux 本地文件系统和必要图形协议的 GUI 文件浏览与后续文件操作能力。
- 非目标：不支持 GVFS、SFTP、SMB、MTP、网络邻居、云盘、桌面图标、系统 MIME 默认应用数据库或桌面服务集成。

## 2. 功能边界

- 核心功能：本地目录浏览、访问历史后退/前进、隐藏文件过滤、无系统边框窗口、窗口拖拽/关闭/最小化/最大化/边缘缩放、侧边栏本地导航、可编辑地址栏、网格文件图标、基础文件类型展示；后续逐步补充安全写操作和外部二进制适配。
- 不支持功能：命令行/TUI、系统桌面服务、网络文件系统、桌面注册接口。
- 关键对象：本地路径、目录条目、文件类型、隐藏状态、文件元数据。
- 关键状态：当前目录、目录条目列表、隐藏文件显示开关、状态消息。

## 3. 关键场景

| 场景 | 用户目标 | 成功标准 | 异常/边界 |
|------|----------|----------|-----------|
| 浏览本地目录 | 查看当前目录内容并进入子目录 | GUI 以暗色文件管理器布局展示目录条目，目录优先排序，可进入子目录并通过后退/前进访问历史位置 | 路径不存在或不可读时展示错误状态 |
| 切换隐藏文件 | 查看或隐藏点号开头文件 | 开关后刷新列表 | 不读取桌面配置服务 |
| 后续打开文件 | 通过用户配置的外部命令处理文件 | 缺命令时功能降级，不影响浏览 | 不使用 `xdg-open` 或 MIME 服务 |

## 4. 核心流程

```text
1. 启动 GUI。
2. 从当前工作目录扫描本地目录。
3. 展示无系统边框窗口、侧边栏、路径栏和网格文件条目。
4. 用户进入目录、后退/前进访问历史位置或切换隐藏文件。
5. 扫描失败时在窗口状态区域展示错误。
```

## 5. 产品规则

- 权限规则：首版只读浏览不修改用户文件；后续写操作必须有明确边界和失败恢复策略。
- 状态流转：目录切换后重新扫描；扫描成功更新条目，扫描失败清空条目并展示错误。
- 异常处理：缺失路径、不可读路径和 metadata 读取失败必须显式反馈。
- 兼容约束：同一二进制应可在 X11 和 Wayland 会话运行，不分别编译。
- 用户可见行为：默认不显示隐藏文件；普通文件暂不自动打开；窗口英文标题和 Linux app_id 使用 `File`；左侧拖拽标题栏显示 `文件`；窗口/任务栏图标使用 `icons/fs.svg`；地址栏左侧只有后退和前进按钮，分别使用 `icons/left.svg` 与 `icons/right.svg`，无历史时置灰不可点；侧边栏只显示主文件夹、根目录和用户家目录中实际存在的常见目录，导航项使用本地 SVG 图标并保持左对齐、垂直居中；侧边栏顶部和地址栏右侧空白区可拖拽窗口；窗口四边和四角可拖动缩放；地址栏可编辑并回车访问路径。
- 外框圆角策略：只接受窗口管理器原生支持的外框圆角；当前 Linux iced/winit 未提供可用接口时，不使用透明窗口或内容裁剪模拟圆角。

## 6. 非功能要求（按需）

- 性能：目录扫描优先同步简单实现；大目录性能后续专项验证。
- 可用性：GUI 是唯一交互形态，不提供命令行/TUI。
- 安全：不依赖系统桌面服务；后续外部命令必须使用 argv 直接调用，不经 shell。
- 兼容性：使用 `wgpu` 渲染；同一二进制启用 X11 和 Wayland。
